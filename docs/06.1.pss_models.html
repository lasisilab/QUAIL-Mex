Overview

SHOULD consider removing data point with household size of 40 (vecindad), as it may be distorting results.

This script fits a series of linear regression models predicting Perceived Stress Scale total score (PSS_TOTAL). Each model shares the same set of core control variables; one water-related predictor is added per model. Models are defined in a named list (model_specs).

Role Variables
Outcome PSS_TOTAL (continuous, 0–56)
Core (every model) Age (D_AGE), SES composite (SES_SC_Total), Household size (D_HH_SIZE)
Augmenting (one per model) HWISE total · individual HWISE items · hours water supply/week · continuous/intermittent supply · WS/WI neighborhood · neighborhood comparison · time handling water

MISSING_CODES <- c(999, 888, 777, 666, 555, 444, 333)

# Replace sentinel codes with NA for numeric variables
clean_num <- function(x) {
  x_num <- suppressWarnings(as.numeric(as.character(x)))
  ifelse(x_num %in% MISSING_CODES, NA_real_, x_num)
}

    # ── Core predictors ────────────────────────────────────────────────────────
    D_AGE        = clean_num(D_AGE),

    D_HH_SIZE    = clean_num(D_HH_SIZE),

    # ── HWISE ──────────────────────────────────────────────────────────────────
    HW_TOTAL     = clean_num(HW_TOTAL),
    across(all_of(hwise_items), clean_num),

    # ── Hours of water supply per week ─────────────────────────────────────────
    HRS_PER_WEEK = clean_num(MX5_WFREQ_HRS_PER_WEEK),

    # ── Continuous vs. intermittent supply ─────────────────────────────────────
    # Accepts numeric (0/1) or text ("Continuo"/"Intermitente")
    CONT_INT = {
      raw <- as.character(MX3_CONT_INT)
      raw_num <- suppressWarnings(as.numeric(raw))
      factor(case_when(
        raw_num %in% MISSING_CODES                         ~ NA_character_,
        raw_num == 0                                       ~ "Continuous",
        raw_num == 1                                       ~ "Intermittent",
        str_detect(str_to_lower(raw), "contin")            ~ "Continuous",
        str_detect(str_to_lower(raw), "inter")             ~ "Intermittent",
        is.na(raw) | str_trim(raw) == "" | raw == "NA"    ~ NA_character_,
        TRUE                                               ~ raw
      ), levels = c("Continuous", "Intermittent"))  # Continuous = reference
    },

    # ── WS/WI neighborhood classification ─────────────────────────────────────
    WS_LOC = factor(W_WS_LOC),

    # ── Neighborhood water quality comparison (Q28) ────────────────────────────
    # 0 = Better (reference), 1 = Same, 2 = Worse
    NB_COMP = factor(
      case_when(
        clean_num(MX28_WQ_COMP_CAT) == 0 ~ "Better",
        clean_num(MX28_WQ_COMP_CAT) == 1 ~ "Same",
        clean_num(MX28_WQ_COMP_CAT) == 2 ~ "Worse",
        TRUE                             ~ NA_character_
      ),
      levels = c("Better", "Same", "Worse")
    ),

    # ── Time handling water (Q19) ──────────────────────────────────────────────
    # 0 = No (reference), 1 = Yes
    TIME_WATER = factor(
      case_when(
        clean_num(MX19_TIME_CAT) == 0 ~ "No",
        clean_num(MX19_TIME_CAT) == 1 ~ "Yes",
        TRUE                          ~ NA_character_
      ),
      levels = c("No", "Yes")

Each augmented model is compared to M0 re-fit on the same complete cases, using an F-test for added predictors. Models with different N from M0 (due to missing data in the augmenting predictor) are flagged.

comparison <- map_dfr(names(fitted_models), function(nm) {
  mod  <- fitted_models[[nm]]
  spec <- model_specs[[nm]]
  fit  <- glance(mod)
  n_mod <- nobs(mod)

  base_row <- tibble(
    model_id     = nm,
    label        = spec$label,
    group        = spec$group,
    n            = n_mod,
    R2           = round(fit$r.squared,     3),
    adj_R2       = round(fit$adj.r.squared, 3),
    delta_adj_R2 = NA_real_,
    AIC          = round(fit$AIC, 1),
    BIC          = round(fit$BIC, 1),
    F_stat       = NA_real_,
    F_df1        = NA_integer_,
    F_df2        = NA_integer_,
    F_p          = NA_real_,
    note         = ""
  )

  if (nm == "core") return(base_row)

  # Refit M0 on same rows as augmented model
  rows_used <- as.integer(rownames(model.frame(mod)))
  m0_sub    <- lm(core_formula, data = df[rows_used, ])
  lrt_tab   <- anova(m0_sub, mod)
  f_stat    <- lrt_tab$F[2]
  f_df1     <- lrt_tab$Df[2]
  f_df2     <- lrt_tab$Res.Df[2]
  f_p       <- lrt_tab$`Pr(>F)`[2]

  base_row %>% mutate(
    delta_adj_R2 = round(fit$adj.r.squared - glance(m0_sub)$adj.r.squared, 3),
    F_stat       = round(f_stat, 2),
    F_df1        = as.integer(f_df1),
    F_df2        = as.integer(f_df2),
    F_p          = round(f_p, 4),
    note         = if_else(n_mod != nobs(fitted_models[["core"]]),
                           "N differs from full M0", "")
  )
})

Interpreting the Results

What each coefficient means

For each augmented model, the β coefficient for the water-related predictor is the expected change in PSS_TOTAL for a one-unit increase in that predictor, holding Age, SES, and Household size constant. A positive β indicates higher water burden associates with higher perceived stress.

95% CI: if the interval does not cross zero, the association is significant at α = 0.05. Wide intervals signal imprecision — check your N in that model.

Model fit criteria

Criterion Prefer
Adj. R² Higher (more variance explained, penalized for parameters)
Δ Adj. R² Positive values mean the water predictor explains additional variance
AIC / BIC Lower. If AIC/BIC of an augmented model > M0’s AIC/BIC, the predictor adds noise
F-test p < 0.05 suggests the augmenting predictor(s) jointly improve fit

Core variable benchmarks (M0)

Check M0 first:

  • SES_SC_Total: expect a negative β (higher SES → lower stress). If positive or near-zero, investigate SES operationalization.
  • D_AGE: typically a modest association (positive or U-shaped).
  • D_HH_SIZE: larger households may share water burden — direction is theoretically ambiguous.

HWISE Total vs. Individual Items

  • HWISE total (HW_TOTAL) provides a summary test: does overall water insecurity predict stress beyond demographics?
  • Individual item models reveal which dimensions drive stress (e.g., worry vs. physical access). Interpret items one at a time — they are intercorrelated components of the same scale. Do not enter all 12 items in a single model without addressing multicollinearity (VIF > 5 is a concern; use car::vif()).

Categorical predictors

Limitations

  1. Cross-sectional design: Predictors and outcome were measured at the same time point. Temporal precedence cannot be established — stress could precede and bias water-related perceptions (reverse causality).

  2. Common method bias: Both PSS and most water variables are self-reported. Shared measurement method can inflate associations regardless of any true relationship.

  3. Listwise deletion: Each model drops participants missing any included variable. Effective N varies across models, complicating direct comparisons. If missingness is informative (e.g., more common in extreme SES groups), results may be biased.

  4. Multiple testing: Testing ~18 separate models on the same outcome and sample substantially increases the familywise error rate. All findings should be treated as exploratory and replicated in independent samples.

  5. HWISE item multicollinearity: Individual items are components of the total score and are highly intercorrelated. Running 12 item models is exploratory; enter items individually, not jointly, without a specific dimensionality hypothesis.

  6. HWISE response scale treated as interval: Individual HWISE items use 4-point response options (0 = never, 1 = rarely, 2 = sometimes, 3 = often/always) and are entered as numeric predictors, assuming equal spacing between categories. This is consistent with prior HWISE validation literature and standard practice for Likert-type items in regression, but the interval assumption is unverified. Sensitivity analyses using dummy-coded categories could assess whether results are robust to this assumption.

  7. Unmeasured confounders: Violence exposure, family conflict, financial shocks, mental health history, and healthcare access are not captured and may confound the water–stress relationship.

  8. Generalizability: Findings reflect a specific sample from colonias in one urban context. Caution is warranted when extrapolating to other settings, water systems, or populations.

  9. PSS as a state measure: PSS captures perceived stress over the past month. Acute events unrelated to water may dominate PSS scores, diluting water-specific associations.

  10. Seasonal conflation: Data from two seasons may blend seasonal within-person change with stable between-person differences. Include SEASON as a covariate or stratify in confirmatory analyses.

  11. Linear assumption: Linear regression assumes a constant marginal relationship between predictors and PSS. Non-linear or threshold effects (e.g., stress only rising above a certain HWISE score) would be missed. Check with splines or polynomial terms if diagnostics suggest non-linearity.
